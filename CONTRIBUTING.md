# Contributing to Conscious-Bridge-Law

We welcome your contributions! Here are the guidelines:

## 1. Reporting Bugs (Issues)
- Open a new Issue when you find a bug.
- Provide a clear and detailed description.
- Include a reproducible example if possible.

## 2. Developing New Features
1. Open an Issue to discuss the idea first.
2. Create a new branch with a descriptive name (feature-<feature_name>).
3. Add tests for the new feature.
4. Submit a Pull Request with documentation of your changes.

## 3. Standards
- Follow the existing code style.
- Document any new features.
- Ensure all tests pass before submitting.

## 4. Communication
- General questions: use GitHub Discussions or Issues.
- Suggestions for improvements are welcome.
